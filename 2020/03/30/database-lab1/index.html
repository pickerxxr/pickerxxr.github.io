数据库及实现 上机实验报告3

刘有哲 18307100052

一、 实验目的

  1. 了解SQL Server 数据库的逻辑结构和物理结构的特点。
  2. 学会使用SQL Server 管理平台对数据库进行管理。
  3. 学会使用 Transact-SQL语句对数据库进行管理。

二、 实验环境

  • 软件配置： Microsoft SQL Server 2008 R2
  • 操作平台： SQL Server Management Studio
  • 系统类型： Windows 10

三、 实验内容

Part Ⅰ

  1. SQL Server 管理平台中创建数据库。

    （1）运行SQL Server管理平台，在管理平台中的对象资源管理器中展开服务器。

    （2）

  1. 选择studentsdb 数据库，在其快捷菜单中选择“新建数据库”菜单项，在新进数据库对话框的数据库名称文本框中输入学生管理数据库名studentsdb，单机确认按钮。

  1. 打开属性，在‘文件’选项卡中的数据库文件列表中修改studentsdb行数据文件列表中修改studentsdb行数据文件的‘分配的空间’为5mb。指定‘最大文件大小’为200MB。修改studentdb数据库的日治文件的大小每次填满是自动递增10%。

  2. 单机新建查询打开查询编辑器窗口，在查询编辑器窗口中使用Transact-SQL语句CREAT DATABASE　创建studb数据库, 然后通过系统存储过程sp_helpdb查看系统中的数据库信息.

    • 其中第二个语句返回的结果如下：

      ​

  3. 在查询编辑器中使用Transact-SQL语句ALTER DATABASE修改studb数据库的设置, 指定行数据文件大小为5MB,最大文件为10MB.

    • 在图形化界面看，结果显然已经更改。

  4. 在查询编辑器中为studb数据库增加一个日志文件,命名为studb_Log2,大小为5MB,最大文件为10MB.

  5. 使用SQL Server管理平台将studb数据库的名称改为student_db.

    • 下图中可以看到，结果已经更改

  6. 在SQL Server管理平台中删除student_db数据库,或者使用Transact-SQL语句DROP DATABASE删除student_db数据库.
